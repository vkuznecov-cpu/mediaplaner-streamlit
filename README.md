# Media Planner

Streamlit-приложение для расчета медиаплана по месяцам с поддержкой пресетов `E-com`, `DIY` и `Недвижимость`.

## Что умеет

- расчет среднего месяца и помесячного медиаплана;
- настройка коэффициентов сезонности: спрос, AOV, кастомные наборы, медийные хвосты;
- настройка НДС и АК;
- сводные таблицы и диаграммы для проверки расчетов;
- экспорт и импорт состояния проекта через JSON;
- экспорт в Excel и шаблоны.

## Структура проекта

- `app.py` — основное приложение;
- `requirements.txt` — Python-зависимости;
- `templates/` — Excel-шаблоны;
- `assets/` — изображения и интерфейсные ассеты;
- `.streamlit/config.toml` — локальная конфигурация Streamlit, включая фиксированную темную тему.

## Требования

- Python 3.10+;
- зависимости из `requirements.txt`.

## Установка

```powershell
python -m venv .venv
.venv\Scripts\activate
python -m pip install --upgrade pip
pip install -r requirements.txt
```

## Запуск

```powershell
python -m streamlit run app.py
```

Обычно приложение будет доступно по адресу `http://localhost:8501`.

## Что публиковать в GitHub

Имеет смысл включать в репозиторий:

- `app.py`
- `README.md`
- `requirements.txt`
- папки `assets/` и `templates/`
- `.streamlit/config.toml`

Не стоит включать:

- `__pycache__/`
- `.venv/`
- логи Streamlit
- временные Excel-файлы
- локальные backup-файлы `app.py`
- пользовательские JSON-выгрузки с рабочими данными

## Быстрая проверка перед публикацией

1. Установить зависимости: `pip install -r requirements.txt`
2. Проверить синтаксис: `python -m py_compile app.py`
3. Запустить приложение: `python -m streamlit run app.py`
4. Проверить основные вкладки: `Настройки расчета`, `Медиаплан`, `Диаграммы`

## Заметки

- Excel-шаблоны в `templates/` можно обновлять отдельно без изменения общей структуры приложения.
- Темная тема зафиксирована на уровне Streamlit-конфига и не зависит от темы браузера пользователя.
